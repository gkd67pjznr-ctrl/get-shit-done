# Plan 36-02: End-to-End Smoke Test — Gate Cycle Verification

## Goal

Demonstrate the complete gate observability cycle: a real task triggers the `gsd-run-gates.cjs` hook, which writes entries to `gate-executions.jsonl`, and the `/api/gate-health` endpoint correctly reflects those new entries. This is the final verification that GATE-05 and GATE-06 are satisfied.

## Context

### Prerequisites (from 36-01)

- `aggregateGateHealth()` correctly reads real `gate-executions.jsonl` entries (verified in 36-01)
- `/api/gate-health` endpoint returns the aggregated data (verified in 36-01)
- `gate-executions.jsonl` already has 30+ real entries from phases 34–35

### What This Plan Demonstrates

GATE-06 requires: "Running `/gsd:quick` produces verifiable gate execution entries."

The mechanism is already wired from Phase 35:
- `gsd-run-gates.cjs` is registered in `.claude/settings.json` as a PostToolUse hook for `Write|Bash`
- Every `npm test` / `vitest` / `node --test` Bash call triggers `test_gate`
- Every Write to a `.ts`/`.js`/`.cjs`/`.mjs` file triggers `diff_review`

Running any test command or writing a code file during this session will trigger the hook. This plan captures the before/after counts and verifies the delta.

### Quality Level

Check that `quality.level` in `.planning/config.json` is `standard` or `strict` — not `fast`. If it is `fast`, the hook exits early and no entries are written. Read the config first.
