# Plan 33-02: Surface Pending Suggestions at Session Start

## Goal

When `suggest_on_session_start: true` is set in config, pending entries from `suggestions.json` are presented to the user at the start of each session, showing the target skill name and proposed change clearly enough to act on.

## Context

- `suggest_on_session_start: true` is already set under `adaptive_learning.integration` in `.planning/config.json`.
- `suggestions.json` at `.planning/patterns/suggestions.json` currently has one pending entry: `sug-1773200068-001` for `code-review` skill, category `process.regression`, created 2026-03-11. This will auto-expire around 2026-04-10 — plan must be executed before that date to observe the live entry.
- `gsd-recall-corrections.cjs` is the existing SessionStart hook at `.claude/hooks/gsd-recall-corrections.cjs`. It already injects a `<system-reminder>` block and handles token budgeting. Suggestion surfacing must be added to this same hook to keep session-start output consolidated in one block.
- The hook uses CommonJS `require()` and handles errors silently. Additions must follow the same pattern.
- The `auto_dismiss_after_days: 30` setting means `analyzePatterns()` (run in 33-01) auto-dismisses expired suggestions before analysis. The surfacing code must read `status: 'pending'` entries only.
