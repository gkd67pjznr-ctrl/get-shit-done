# Plan 34-01: Implement Skill Refinement Flow

## Goal

Create the mechanics that allow a user to accept or dismiss a pending suggestion. Accepting must write the sample correction text into the target SKILL.md as a new pattern note and commit the change. Dismissing must remove the suggestion entry from suggestions.json without touching any skill file.

## Context

- `suggestions.json` lives at `.planning/patterns/suggestions.json`
- The pending suggestion is `sug-1773200068-001` targeting `code-review` in category `process.regression`
- `code-review` SKILL.md is at `.claude/skills/code-review/SKILL.md`
- `retire.cjs` at `.claude/hooks/lib/retire.cjs` handles marking suggestions as `refined` and retiring matching corrections — it is already written and tested
- `analyze-patterns.cjs` marks accepted status as `'accepted'` in the cooldown guard, and `'refined'` after retire runs
- The recall hook references `/gsd:refine-skill <skill-name>` as the user-facing command — this command does not yet exist

## Design

### Accept path

1. Read the pending suggestion from `suggestions.json` by skill name (or suggestion ID)
2. Read the target `SKILL.md` file
3. Append a `## Learned Patterns` section (or extend it if it exists) with each `sample_corrections` item as a bullet under the relevant heading
4. Write the updated SKILL.md atomically
5. Stage and commit with message `refine(skill/<name>): apply correction pattern from suggestion <id>`
6. Call `retireByCategory(category, suggestionId)` from `retire.cjs` to mark the suggestion `refined` and retire matching corrections/preferences
7. Report success to stdout

### Dismiss path

1. Read `suggestions.json`
2. Find the suggestion by ID (or skill name for the pending one)
3. Set `status: 'dismissed'`, `dismissed_at: <ISO timestamp>`, `dismiss_reason: 'user_dismissed'`
4. Write `suggestions.json` atomically (tmp+rename pattern)
5. Report success to stdout (no git commit, no skill file change)

<action>
Create `.claude/hooks/lib/refine-skill.cjs` as a CJS module. It must export two functions:

**`acceptSuggestion({ suggestionId, cwd })`**

1. Read `.planning/patterns/suggestions.json`
2. Find the suggestion where `s.id === suggestionId` and `s.status === 'pending'`. If not found, return `{ ok: false, reason: 'not_found' }`.
3. Resolve the skill file path: `path.join(cwd, '.claude', 'skills', suggestion.target_skill, 'SKILL.md')`. If the file does not exist, return `{ ok: false, reason: 'skill_file_missing', path: skillPath }`.
4. Read the SKILL.md content.
5. Build the content to append:
   - Check if the file already contains `## Learned Patterns` — if not, append it as a new H2 section.
   - Under `## Learned Patterns`, append one bullet per entry in `suggestion.sample_corrections` (skip empty strings). Format: `- [<category>] <correction_text>`
   - Separate new content from existing content with a blank line before the section if adding it fresh.
6. Write the updated SKILL.md using `fs.writeFileSync` (not atomic tmp for this file — it is tracked by git and atomic rename would confuse git's index; direct write is correct here).
7. Stage the SKILL.md via `child_process.execSync('git add -f ' + skillPath, { cwd })`.
8. Commit via `child_process.execSync('git commit -m "refine(skill/' + suggestion.target_skill + '): apply correction pattern from suggestion ' + suggestionId + '"', { cwd })`.
9. Call `retireByCategory(suggestion.category, suggestionId, { cwd })` from `./retire.cjs`.
10. Return `{ ok: true, skillPath, committed: true }`.

Wrap the entire function body in try/catch. On catch, return `{ ok: false, reason: 'error', error: e.message }`.

**`dismissSuggestion({ suggestionId, cwd })`**

1. Read `.planning/patterns/suggestions.json`.
2. Find the suggestion where `s.id === suggestionId` and `s.status === 'pending'`. If not found, return `{ ok: false, reason: 'not_found' }`.
3. Set `s.status = 'dismissed'`, `s.dismissed_at = new Date().toISOString()`, `s.dismiss_reason = 'user_dismissed'`.
4. Write the updated `suggestions.json` atomically using tmp+rename (same pattern used in `analyze-patterns.cjs`).
5. Return `{ ok: true, dismissed: true }`.

Wrap the entire function body in try/catch. On catch, return `{ ok: false, reason: 'error', error: e.message }`.

**CLI invocation** (when `require.main === module`):

```
node refine-skill.cjs accept <suggestionId> [cwd]
node refine-skill.cjs dismiss <suggestionId> [cwd]
```

Print JSON result to stdout. Exit 0 on success, exit 1 on `ok: false`.

File header comment: `// Skill refinement library: accept or dismiss a pending suggestion.`

Zero external dependencies — built-in Node.js only plus require('./retire.cjs') and require('child_process').
