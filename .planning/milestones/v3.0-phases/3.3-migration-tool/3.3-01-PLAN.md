<feature>
  <name>Migration Tool (migrate.cjs)</name>
  <files>get-shit-done/bin/lib/migrate.cjs, get-shit-done/bin/gsd-tools.cjs, tests/migrate.test.cjs</files>
  <behavior>
    The migrate command inspects a project's `.planning/` directory against the current GSD spec and reports or applies additive-only structural changes.

    **inspectLayout(cwd)** — shared private function:
    - Checks for `.planning/` root dir existence
    - Checks for `.planning/phases/` directory
    - Checks for `.planning/config.json` file
    - Returns `{ layout: string, changes: Change[], summary: { layout, changes_needed, manual_actions } }`
    - Each change is `{ type: 'create_dir'|'create_file'|'manual_action', path: string, reason: string, content?: string }`
    - `manual_action` items are reported but NOT applied (e.g., ROADMAP.md, STATE.md, PROJECT.md which need user content)

    **cmdMigrateDryRun(cwd, opts, raw)** — MIGR-01:
    - Calls `inspectLayout(cwd)`
    - Outputs JSON: `{ dry_run: true, changes: Change[], summary: { layout, changes_needed, manual_actions } }`
    - Raw value: `'up-to-date'` if 0 changes, `'changes-needed'` if > 0

    **cmdMigrateApply(cwd, opts, raw)** — MIGR-02, MIGR-03:
    - Calls `inspectLayout(cwd)` for change list
    - For each `create_dir` change: `if (!fs.existsSync(path)) fs.mkdirSync(path, { recursive: true })`
    - For each `create_file` change: `if (!fs.existsSync(path)) fs.writeFileSync(path, content)`
    - Skips `manual_action` items (report only)
    - Writes `migrate-undo.json` at `.planning/migrate-undo.json` via `fs.writeFileSync` (overwrite, NOT append)
    - Manifest format: `{ applied: ISO8601, actions: [{ type: 'created_dir'|'created_file', path: string }] }`
    - Outputs JSON: `{ applied: true, actions_taken: N, actions: Action[], manifest_path: string, skipped_manual: ManualAction[] }`
    - Raw value: `'already-up-to-date'` if 0 actions, `'applied-N-changes'` if > 0
    - Idempotency: re-running when everything exists takes 0 actions, writes empty-actions manifest

    Cases (input -> expected output):
    - Empty `.planning/` (no config.json, has phases/) -> dry-run reports 1 change (create config.json), 3 manual actions (ROADMAP.md, STATE.md, PROJECT.md)
    - No `.planning/` at all -> error exit (cannot migrate without root dir)
    - Fully-configured legacy project -> dry-run reports 0 changes (already up-to-date)
    - Fully-configured concurrent project -> dry-run reports 0 changes
    - Apply on empty `.planning/` -> creates config.json, writes manifest with 1 action
    - Apply twice -> second run: 0 actions, manifest overwritten with empty actions array
  </behavior>

    4. Verify all tests pass and full regression suite passes

    - Private `inspectLayout(cwd)` function
    - Exported `cmdMigrateDryRun(cwd, opts, raw)` and `cmdMigrateApply(cwd, opts, raw)`

    Router wiring:
    - Add `const migrate = require('./lib/migrate.cjs');` at top of gsd-tools.cjs (after debt require)
    - Add `case 'migrate':` block using `args.includes('--dry-run')` and `args.includes('--apply')` for flag detection (NOT positional args[1] check, per RESEARCH.md Pitfall 1)
    - Default: `error('Usage: migrate --dry-run | migrate --apply')`

    config.json default content when creating:
    ```json

    ```
    (Same minimal defaults as loadConfig fallback — do NOT call config-ensure-section internally, keep module boundaries clean)

    Migration spec checks (in order):
    1. `.planning/` must exist (error if not — cannot migrate nothing)
    2. `.planning/phases/` directory — create_dir if missing
    3. `.planning/config.json` — create_file if missing (with default content above)
    4. `.planning/ROADMAP.md` — manual_action if missing (requires user content)
    5. `.planning/STATE.md` — manual_action if missing (requires user content)
    6. `.planning/PROJECT.md` — manual_action if missing (requires user content)

    Anti-patterns to avoid (from RESEARCH.md):
    - Do NOT modify existing files (additive-only per MIGR-02)
    - Do NOT use appendFileSync for manifest (use writeFileSync per Pitfall 2)
    - Do NOT duplicate inspection logic between dry-run and apply (share inspectLayout per Pitfall 3)
    - Do NOT check config.json section contents (that is config-ensure-section's job per Open Question 1)
    - Do NOT add --milestone flag (migration is project-level, not milestone-scoped per Research)
