### Test Implementation Pattern
The existing test file uses a "simulated" pattern -- it does NOT import from `bin/install.js`. Instead it re-implements small slices of logic inline. New test implementations should follow this same pattern rather than trying to import and invoke the real installer.

```javascript
// Pattern: simulate the runtime guard
test('skills not copied for non-claude runtimes', () => {
  const { tmpDir } = createInstallerTempDir();
  const runtime = 'opencode';
  const skillsDest = path.join(tmpDir, 'skills');

  // Simulate: installer only copies skills when runtime === 'claude'
  if (runtime === 'claude') {
    // would copy skills here
  }

  const installed = fs.readdirSync(skillsDest).filter(e =>
    fs.statSync(path.join(skillsDest, e)).isDirectory()
  );
  assert.strictEqual(installed.length, 0, 'No skills should be installed for non-claude runtime');
  cleanup(tmpDir);
});
```

## Common Pitfalls

### Pitfall 1: Forgetting the dist/ Copy
**What goes wrong:** Editing `hooks/phase-boundary-check.sh` but not rebuilding `hooks/dist/`
**How to avoid:** Run `npm run build` after editing, OR note that dist/ is gitignored and only source matters for git commits. The build step (`scripts/build-hooks.js`) copies hooks to dist/.

### Pitfall 2: VALIDATION.md Task IDs Mismatch
**What goes wrong:** Updating frontmatter but leaving task rows as "pending"
**How to avoid:** Update BOTH the frontmatter AND the per-task status column AND the sign-off checklist.

### Pitfall 3: Coverage Summary Math
**What goes wrong:** Updating checkboxes but forgetting the coverage summary at bottom of REQUIREMENTS.md
**How to avoid:** Lines 149-153 have explicit counts that must be updated: Complete 36->39, Pending 3->0.

### Pitfall 4: Test Stubs -- Over-Engineering
**What goes wrong:** Trying to actually invoke `install()` from install.js, which requires complex mocking of process.argv, readline, etc.
**How to avoid:** Follow the existing simulated pattern. The stubs say "requires invoking install()" but the established pattern in the file is to simulate the logic, not call the real function.
