In `cmdPhaseComplete` function, after the existing directory scan (line 799, after `} catch {}`), add a ROADMAP.md fallback:

```javascript
// Fallback: check ROADMAP.md for higher phases not yet completed
if (isLastPhase && fs.existsSync(roadmapPath)) {
  try {
    const roadmapContent = fs.readFileSync(roadmapPath, 'utf-8');
    const phasePattern = /[-*]\s*\[([x ])\]\s*\*{0,2}Phase\s+(\d+[A-Z]?(?:\.\d+)*):/gi;
    let m;
    while ((m = phasePattern.exec(roadmapContent)) !== null) {
      const isComplete = m[1].toLowerCase() === 'x';
      const foundPhaseNum = m[2];
      if (!isComplete && comparePhaseNum(foundPhaseNum, phaseNum) > 0) {
        isLastPhase = false;
        break;
      }
    }
  } catch {
    // If ROADMAP.md can't be read, keep the directory-based result
  }
}
```

Key implementation details:
- The regex matches `- [ ] **Phase 9:` or `- [x] **Phase 8:` formats (with or without bold markers)
- Only sets `isLastPhase = false` for phases that are BOTH higher-numbered AND not completed (`[ ]` not `[x]`)
- The `comparePhaseNum` import is already available in phase.cjs (line 7)
- `roadmapPath` is already defined earlier in `cmdPhaseComplete` (line 698)
- Wrapped in try/catch so a malformed ROADMAP.md doesn't crash phase completion
