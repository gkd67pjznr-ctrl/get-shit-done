### Integration Issue 1 (HIGH): cmdMilestoneComplete phasesDir

`milestone.cjs` line 87 hardcodes `phasesDir = path.join(cwd, '.planning', 'phases')`. For the active v3.0 milestone-scoped layout (`concurrent: true`), phases reside at `.planning/milestones/v3.0/phases/`. The workspace ROADMAP finalization silently produces no checkbox updates because it iterates an empty directory.

**Fix:** Thread `milestoneScope` through `cmdMilestoneComplete` and use `planningRoot()` for `phasesDir`.

### Integration Issue 2 (MEDIUM): execute-plan.md milestone scope

`execute-plan.md` line 383 calls `roadmap update-plan-progress "${PHASE}"` where PHASE is extracted as `3.1-01` (plan format, not phase format). No `--milestone` flag is passed, so updates write to root `.planning/ROADMAP.md` instead of the milestone workspace.

**Fix:** Extract phase-only portion from PHASE variable; thread `MILESTONE_FLAG` from execute-phase.md into executor subagent context.
