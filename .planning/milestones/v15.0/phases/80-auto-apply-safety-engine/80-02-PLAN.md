# Plan 80-02: Auto-Apply Wiring — Config Key, SessionEnd Hook, Session-Start Notification, and Integration Tests

## Goal

Wire `auto-apply.cjs` into the live hook pipeline, add the `auto_apply: false` config key to `config.json` and `install.cjs` migration, extend the session-start hook to surface auto-apply notifications, and write integration tests covering all five safety gates.

## Context

- Plan 80-01 creates `.claude/hooks/lib/auto-apply.cjs` with `runAutoApply({ cwd })`. This plan wires it into the runtime.
- SessionEnd hook at `.claude/hooks/gsd-analyze-patterns.cjs` calls `analyzePatterns()` then writes the watermark. `runAutoApply` must be called after `analyzePatterns` completes.
- Session-start hook at `.claude/hooks/gsd-recall-corrections.cjs` outputs the recall header and pending suggestions. Extend it to also output a brief auto-apply summary when new entries exist in `auto-applied.jsonl` since the last session.
- `project-claude/install.cjs` handles config migration. It must inject `auto_apply: false` under `adaptive_learning` if absent.
- Tests use Node.js built-in test runner (`node:test`). Import pattern: `const { describe, it } = require('node:test'); const assert = require('node:assert/strict');`.
- Tests are standalone scripts in `tests/` — no build step needed. Run with `node --test tests/auto-apply.test.cjs`.
- Silent failure convention: hooks catch all errors and exit 0.
- `/gsd:settings` toggle for `auto_apply` is **deferred to Phase 81**. CONTEXT.md notes this as a desired UX feature but it was not included in Phase 80's scope. AUTO-03 coverage in this phase is limited to the audit log entries and the session-start notification. The settings command toggle will be addressed when Phase 81 (revert command) is planned, since both touch the same `/gsd:settings` surface.

**Helper function** (copy the fixture pattern from `tests/patterns.test.cjs`):

```javascript
function createFixture(opts = {}) {
  const tmp = fs.mkdtempSync(path.join(os.tmpdir(), 'gsd-aa-'));
  const patternsDir = path.join(tmp, '.planning', 'patterns');
  const skillDir = path.join(tmp, '.claude', 'skills', opts.skillName || 'typescript-patterns');
  fs.mkdirSync(patternsDir, { recursive: true });
  fs.mkdirSync(skillDir, { recursive: true });
  // Init git repo
  execSync('git init && git config user.email t@t.com && git config user.name T', { cwd: tmp });
  // Write SKILL.md with enough content to avoid size gate on small additions
  fs.writeFileSync(path.join(skillDir, 'SKILL.md'), opts.skillContent || ('# Skill\n\n' + 'Content '.repeat(50) + '\n'));
  execSync('git add -A && git commit -m "init"', { cwd: tmp });
  // Config
  fs.writeFileSync(path.join(tmp, '.planning', 'config.json'),
    JSON.stringify({ adaptive_learning: { auto_apply: opts.autoApply !== undefined ? opts.autoApply : true } }));
  // Suggestion
  const sug = Object.assign({
    id: 'sug-test-001', status: 'pending', target_skill: opts.skillName || 'typescript-patterns',
    confidence: 0.97, category: 'code.wrong_pattern', sample_corrections: ['Use const'],
    source_corrections: []
  }, opts.suggestion || {});
  fs.writeFileSync(path.join(patternsDir, 'suggestions.json'), JSON.stringify({ suggestions: [sug] }));
  fs.writeFileSync(path.join(patternsDir, 'corrections.jsonl'), '');
  return { tmp, patternsDir, skillDir, sug };
}
```

**Test cases to implement:**

1. `describe('CONFIG gate')`:
   - `it('skips all suggestions when auto_apply is false')`: Create fixture with `autoApply: false`. Call `runAutoApply`. Assert `result.reason === 'auto_apply_disabled'` and no `auto-applied.jsonl` is created.
   - `it('skips all suggestions when auto_apply key is absent')`: Write config without `auto_apply` key. Same assertions.

2. `describe('RATE gate')`:
   - `it('skips suggestion when same skill applied within 7 days')`: Write a recent "applied" entry to `auto-applied.jsonl` for the target skill. Call `runAutoApply`. Assert `result.skipped >= 1`. Read `auto-applied.jsonl` — assert a "skipped" entry exists with `gate: 'rate'`.
   - `it('passes suggestion when last apply was more than 7 days ago')`: Write an "applied" entry with timestamp 8 days ago. Call `runAutoApply`. Assert `result.applied >= 1` OR suggestion was not blocked by rate gate (may be blocked by quality gate — assert the skipped entry's gate is NOT 'rate').

3. `describe('QUALITY gate')`:
   - `it('skips high-quality skills')`: Write `skill-metrics.json` with an entry for the target skill that causes `computeAttributionConfidence` to return `'high'` (set `sessionCount >= 10` and `correctionCount > 0` in the metrics). Assert result has "skipped" entry with `gate: 'quality'`.

4. `describe('CONFIDENCE gate')`:
   - `it('skips suggestion with confidence <= 0.95')`: Create fixture with `suggestion.confidence = 0.95`. Assert "skipped" with `gate: 'confidence'`.
   - `it('skips controversial category even with high confidence')`: Create fixture with `suggestion.confidence = 0.99` and `suggestion.category = 'code.over_engineering'`. Assert "skipped" with `gate: 'confidence'`.
   - `it('passes suggestion with confidence > 0.95 and safe category')`: Create fixture with valid confidence and safe category. Verify suggestion is NOT blocked by confidence gate (may proceed to size or apply).

5. `describe('SIZE gate')`:
   - `it('skips suggestion when bullet content adds >= 20% of skill length')`: Create fixture with a short SKILL.md (e.g., 100 chars) and a suggestion with a very long correction string (>20 chars). Assert "skipped" with `gate: 'size'`.

6. `describe('audit log')`:
   - `it('writes applied entry with required fields when all gates pass')`: Create a fixture that passes all gates (short skill relative to small corrections, high confidence, safe category, no recent applies, no high-quality metrics). Call `runAutoApply`. Read `auto-applied.jsonl`. Assert applied entry has all fields: `action`, `suggestion_id`, `skill`, `commit_sha`, `confidence`, `change_percent`, `source_corrections`, `before_diff`, `reversal_instructions`, `timestamp`.
   - `it('reversal_instructions contains commit_sha')`: From the same applied entry, assert `entry.reversal_instructions === 'git revert ' + entry.commit_sha`.
