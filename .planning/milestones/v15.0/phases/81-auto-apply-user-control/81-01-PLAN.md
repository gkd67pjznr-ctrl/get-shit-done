function revertAutoApply({ suggestionId, cwd }) { ... }

Implementation steps:

2. Read `auto-applied.jsonl` from `.planning/patterns/auto-applied.jsonl`. If the file does not exist, return `{ ok: false, reason: 'audit_log_missing' }`.

4. Check if a subsequent line exists with `suggestion_id === suggestionId` and `action === 'reverted'`. If found, return `{ ok: false, reason: 'already_reverted' }`.

5. Extract `commit_sha` from the found entry. If `commit_sha` is empty or missing, return `{ ok: false, reason: 'no_commit_sha' }`.

6. Run `git revert --no-edit <commit_sha>` using `execSync` with `{ cwd: resolvedCwd }`. Wrap in try/catch — if it throws (e.g., conflict), return `{ ok: false, reason: 'git_revert_failed', error: e.message }`.

```js
{
  action: 'reverted',
  suggestion_id: suggestionId,
  skill: foundEntry.skill,
  reverted_commit_sha: commit_sha,
  reverted_at: new Date().toISOString(),
}
```

The `auditPath` is `path.join(resolvedCwd, '.planning', 'patterns', 'auto-applied.jsonl')`.

8. Return `{ ok: true, reverted: true, skill: foundEntry.skill, commit_sha }`.
