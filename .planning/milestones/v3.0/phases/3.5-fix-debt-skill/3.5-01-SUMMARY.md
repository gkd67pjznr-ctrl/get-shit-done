## Decisions Made

- **Inline mini-plan (Approach A):** Orchestrator writes fix plan directly to `.planning/debug/debt-ID-fix/01-PLAN.md` and spawns gsd-executor on the file path, bypassing ROADMAP phase lookup. No plan-checker loop overhead for targeted debt fixes.
- **Rich-description skip path:** In yolo mode, when entry description length >50 chars and non-generic, diagnosis step is skipped. Debt entries from agent auto-logging (Phase 3.4) already carry actionable descriptions; redundant investigation avoided.
- **Scope guard before execution:** If diagnosis reveals fix scope broader than the entry's `component` field (more than 3-4 files), orchestrator presents to user before proceeding, preventing Pitfall 2 (plan execution scope leak).
- **Error recovery table:** Explicit mapping of failure points to status transitions ensures entries never get stuck in `in-progress`. Diagnosis fail -> back to `open`; plan/execution/verification fail -> `deferred`.
