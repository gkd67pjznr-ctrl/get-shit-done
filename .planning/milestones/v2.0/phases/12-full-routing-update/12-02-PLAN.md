  <action>
Create `get-shit-done/references/agent-teams.md` documenting the research finding that Agent Teams is recommended for intra-milestone parallel phases but NOT for inter-milestone concurrency.

Content must cover:

1. **Summary:** Agent Teams is a Claude Code feature for parallel task execution within a single session. It's suitable for running independent phases within a single milestone simultaneously (intra-milestone parallelism, v2.1 scope). It is NOT suitable for inter-milestone concurrency (v2.0 scope).

2. **Why not inter-milestone:** Each concurrent milestone runs in its own Claude Code session. Claude Code session resumption is incompatible with Agent Teams' execution model. The v2.0 approach (workspace isolation + explicit --milestone flag + file-based STATUS.md polling) is the correct pattern for inter-milestone concurrency.

3. **Why yes intra-milestone:** Within a single milestone, independent phases (no dependency overlap, no shared files) can potentially be executed in parallel using Agent Teams. This is deferred to v2.1 (TEAM-02, TEAM-03 in REQUIREMENTS.md).

4. **Architecture note:** The workspace isolation foundation built in v2.0 (planningRoot, detectLayoutStyle, milestoneScope threading) also benefits future Agent Teams integration — each agent thread would operate within the same milestone workspace, using the same planningRoot.
