<objective>
Make the Context7 per-query token cap configurable via quality.context7_token_cap in config. The executor reads this value and passes it as the token limit to Context7 queries, so users can tune query size without code edits.

Purpose: Context7 API quota is a real concern (noted as a blocker in STATE.md). A configurable cap lets users control token consumption per query, defaulting to 2000 tokens — a conservative starting point validated during v1.0 planning.

Output: Updated executor agent instructions referencing configurable token cap, config.cjs already has the default from Plan 01, tests proving the config key is readable and applied.
</objective>
