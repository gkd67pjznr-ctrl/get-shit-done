# Requirements — Milestone v6.0

*Created: 2026-03-10*
*Updated: 2026-03-11 (populated from root REQUIREMENTS.md and phase RESEARCH.md files)*

## Correction Capture (Phase 22)

- [x] **CAPT-01**: Hook-based correction capture detects when user corrects Claude and logs structured entries to corrections.jsonl with scope tagging
- [x] **CAPT-02**: Auto self-diagnosis runs when a correction is captured — Claude analyzes root cause using 14-category two-tier taxonomy with 100-token cap
- [x] **CAPT-03**: Correction entries include `correction_from`, `correction_to`, `diagnosis_category`, `diagnosis_text`, `scope`, and `phase` fields
- [x] **CAPT-04**: JSONL rotation enforces retention_days config — rotates at 1000 lines, archives to dated files

## Preference Learning (Phases 23, 24, 27)

- [x] **PREF-01**: Preference tracking promotes 3+ repeated corrections on the same pattern to a durable preference in preferences.jsonl with confidence scoring
- [x] **PREF-02**: Preferences are scope-aware — tagged with scope (file/filetype/phase/project/global), defaulting to narrowest applicable scope
- [x] **PREF-03**: Cross-project preference inheritance — user-level preferences stored at `~/.gsd/preferences.json`, promoted when a preference appears in 3+ projects
- [x] **PREF-04**: When a preference is baked into a skill refinement, it is retired from the active preference store and recall pipeline (recall exclusion: Phase 24, retirement mechanism: Phase 26)

## Live Recall (Phase 24)

- [x] **RECL-01**: Session-start recall loads relevant corrections and preferences into context, capped at 10 entries (~3K tokens). Phase filtering deliberately omitted per CONTEXT.md (concurrent milestones span many phases)
- [x] **RECL-02**: Live within-session recall cross-references historical corrections during active work so Claude catches itself before repeating known mistakes
- [x] **RECL-03**: Recalled items that have been baked into skills are excluded from recall (skills are the durable layer, recall is the staging area)

## Observer & Pipeline (Phase 25)

- [x] **OBSV-01**: Observer agent replaces the stub — performs session boundary analysis, pattern aggregation, and suggestion generation
- [x] **OBSV-02**: Suggestion pipeline writer populates suggestions.json with candidates when correction patterns cross the min_occurrences threshold
- [x] **OBSV-03**: Observer enforces bounded learning guardrails: 3 corrections min, 7-day cooldown, user confirmation via /gsd:suggest, guardrail violation logging (Phase 25); 20% max change guardrail (Phase 26); 5+ co-activations is skill-creator scope

## Enhanced Analysis (Phase 26)

- [x] **ANLY-01**: Enhanced `/gsd:digest` includes correction analysis section — groups by diagnosis category, shows trends, identifies skills being contradicted
- [x] **ANLY-02**: `/gsd:suggest` provides collaborative skill refinement workflow via AskUserQuestion when corrections consistently contradict a specific skill
- [x] **ANLY-03**: Skill refinement step retires the source corrections/preferences from active recall after baking changes into the skill

## Skill Loading (Phase 27)

- [x] **LOAD-01**: All GSD workflow commands load relevant learned skills into their execution context
- [x] **LOAD-02**: All GSD agents and subagents inherit learned skills in their spawned context
