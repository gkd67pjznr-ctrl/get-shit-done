**3. Sub-check 1: Duplication (same-phase only)**

For each pair of phase files (from SUMMARY_FILES), extract meaningful 5+ consecutive non-blank, non-comment code lines and search for identical blocks in other phase files. Skip lines that are only: `"use strict"`, `const X = require(`, `module.exports`, or comment lines (these are structural boilerplate, not duplicated logic). Minimum match threshold: 5 consecutive identical non-trivial lines.

Output format per finding: `- {SEVERITY}: \`{file_a}\` lines {start}-{end} duplicates \`{file_b}\` lines {start}-{end}. Consider extracting to a shared helper.`

Scope: same-phase files only — do NOT scan the entire codebase. This is a locked user decision.

**4. Sub-check 2: Orphaned Exports (project-wide)**

For each phase file that is `.cjs`, `.js`, or `.ts`:
- Extract exported symbol names (`exports.X`, `module.exports`, `export const/function`)
- For each symbol, grep the entire project (excluding `node_modules`, `.git`, `.planning/`) for imports of that symbol
- Zero matches (excluding the file itself) = orphaned
- **Important:** If the file lives in a `bin/` directory or is named `cli.cjs`, `index.cjs`, or similar known entry point, downgrade to INFO with note: "Possible CLI entry point or public API — verify manually." This is a locked user decision.

Output format: `- {SEVERITY}: \`{file}\` export \`{symbol}\` has no project-internal importers. {suggestion}`

**5. Sub-check 3: Missing Tests**

Same logic as executor sentinel Step 4 (gsd-executor.md lines 148-153) — already proven correct:
- For each phase file: if `.cjs`, `.js`, or `.ts` AND contains `export` keyword
- Check if corresponding `.test.cjs` (or `.test.js`/`.test.ts`) file exists
- Skip files matching `quality.test_exemptions` from config — use `config-get quality.test_exemptions` for the exemption list (same source of truth as executor)
- Missing test = finding

Output format: `- {SEVERITY}: \`{file}\` has no corresponding test file. Expected: \`{expected_test_file}\`.`
