QUALITY_LEVEL=$(node ~/.claude/get-shit-done/bin/gsd-tools.cjs config-get quality.level 2>/dev/null || echo "fast")

**Pre-Task Protocol (runs before each type="auto" task):**

Step 1 — Targeted Codebase Scan (skip if `fast`):
- Identify task domain from `<name>` and `<files>` fields
- Run targeted grep against files relevant to this domain only — NOT the full codebase
- Example: `grep -rn "export.*function\|export const" src/lib/ --include="*.cjs" 2>/dev/null | head -10`
- Rules: scope to directories named in task's `<files>` field or adjacent utility directories; search for existing exported functions matching the task's domain term; if similar function found, evaluate for reuse before writing new code; maximum 10 lines of grep output (truncate with `| head -10`); document reuse decision in commit message; never scan `node_modules`, `.git`, `.planning/`, or archived phase directories

Step 2 — Context7 Lookup (see `<context7_protocol>` section — skip if `fast`)

Step 3 — Test Baseline (skip if `fast`):
- Run `node --test tests/*.test.cjs 2>&1 | tail -3`
- Record pass count before changes
- If baseline is red, document in SUMMARY.md under "Issues Encountered"

**Post-Task Protocol (runs before commit, after all files written):**

Step 4 — Test Gate:
- For each file produced by the task: if file is `.cjs`, `.js`, or `.ts` AND contains `export` keyword, check if corresponding test file exists (same name with `.test.` prefix)
- If test file missing: write it now, before commit
- Test file must exercise exported function(s) with at least one success and one edge case
- Files matching `quality.test_exemptions` (`.md`, `.json`, `templates/**`, `.planning/**`) are exempt
- Mode: standard — new logic files only; strict — always check; fast — skip entirely

Step 5 — Diff Review (standard and strict; skip if `fast`):
- Run `git diff --staged`
- Self-report any of: function/variable names that conflict with existing names in the codebase; logic blocks that duplicate code found in pre-task scan (Step 1); TODO/FIXME comments left in changed lines; error paths with no handling
- Report findings in SUMMARY.md under "Deviations from Plan -> Auto-fixed Issues" or "Issues Encountered"
- Fix before committing if fix scope is within deviation Rules 1-3
- Escalate to Rule 4 if architectural

**Include the gate behavior matrix as a reference table:**

| Gate | fast | standard | strict |
|------|------|----------|--------|
| Pre-task codebase scan | Skip | Run | Run |
| Context7 lookup | Skip | Conditional (new deps only) | Always |
| Test baseline record | Skip | Run | Run |
| Test gate (new logic) | Skip | New `.cjs/.js/.ts` with exports | Always |
| Post-task diff review | Skip | Run | Run |

Use XML tags `<quality_sentinel>` and `</quality_sentinel>` to wrap the section. Use markdown headings within (## level). The section should be approximately 80-120 lines of markdown — enough to be precise, not so long it bloats executor context.
