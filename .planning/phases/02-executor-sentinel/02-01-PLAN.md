Add a new `<context7_protocol>` section to `agents/gsd-executor.md`. Place it AFTER the closing `</tdd_execution>` tag and BEFORE `<task_commit_protocol>`. This position keeps it adjacent to the execution-related sections.

The section must contain these subsections exactly as specified in 02-RESEARCH.md Pattern 4:

**When to Consult Context7:**
- Trigger conditions: task uses an external library from `package.json` not yet used in the current session; task references a specific API method with uncertainty; task involves framework patterns where version-specific behavior matters
- Skip conditions: task modifies only `.md`/`.json`/template files; task uses only Node.js built-ins (`fs`, `path`, `os`, `node:test`, `node:assert`); library already consulted in this session; `quality.level` is `fast`

**How to Call Context7:**
- Step 1: `mcp__context7__resolve_library_id(libraryName, query)` — resolve the library name to a Context7-compatible ID
- Step 2: `mcp__context7__query_docs(libraryId, query)` — query for the specific method/pattern needed (NOT the library overview)

**Token Discipline:**
- Query for the specific method/pattern, not the library overview
- One Context7 query per plan execution maximum
- Extract 3-5 key facts from the response and hold as working state
- Do not quote more than 200 tokens of Context7 response in implementation comments
- If response exceeds ~2,000 tokens, extract only the directly relevant code example

**Mode behavior:**
- `standard`: Call Context7 only when trigger conditions are met
- `strict`: Call Context7 before implementing ANY external library usage, regardless of prior session usage. No exceptions.
- `fast`: Skip entirely (the skip condition for `quality.level` is `fast` handles this)

Use XML tags `<context7_protocol>` and `</context7_protocol>` to wrap the section. Use markdown headings within the section (## level).
