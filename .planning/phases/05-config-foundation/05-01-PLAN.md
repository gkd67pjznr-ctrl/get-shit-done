**A. Config auto-migration (QCFG-02) ã in config.cjs `cmdConfigEnsureSection`:**

Currently, `cmdConfigEnsureSection` returns early with `already_exists` if config.json exists. Change this behavior:

1. If config.json exists, load it and check for missing required sections.
2. Define a `requiredQualityDefaults` object:
   ```javascript
   const requiredQualityDefaults = {
     level: 'fast',
     test_exemptions: ['.md', '.json', 'templates/**', '.planning/**'],
     context7_token_cap: 2000,
   };
   ```
3. If `config.quality` is undefined:
   - Deep-merge quality defaults (with user global defaults taking precedence where they overlap).
   - Set `config.quality = { ...requiredQualityDefaults, ...(userDefaults.quality || {}) }`.
   - Write back to disk.
   - Return `{ created: false, migrated: true, reason: 'quality_section_added' }`.
4. If `config.quality` exists but `context7_token_cap` is missing:
   - Add `config.quality.context7_token_cap = requiredQualityDefaults.context7_token_cap`.
   - Write back to disk.
   - Return `{ created: false, migrated: true, reason: 'context7_token_cap_added' }`.
5. If everything is present, return the existing `{ created: false, reason: 'already_exists' }`.
6. When creating fresh config (no config.json), include `context7_token_cap: 2000` in the hardcoded quality defaults.

**B. Global defaults bootstrap (QCFG-03) ã in config.cjs `cmdConfigEnsureSection`:**

After the existing "Load user-level defaults" block, add bootstrap logic:

1. Determine GSD home: `const gsdHome = process.env.GSD_HOME || path.join(homedir, '.gsd');`
2. Update `globalDefaultsPath` to use `gsdHome` instead of hardcoded `path.join(homedir, '.gsd', 'defaults.json')`.
3. After config.json creation (both new and migrated), if `defaults.json` does NOT exist:
   - Create `gsdHome` directory recursively.
   - Write `defaults.json` with `{ "quality": { "level": "fast" } }`.
   - This bootstraps the global defaults so subsequent projects inherit.
4. Do NOT overwrite existing defaults.json.

Also update `braveKeyFile` path to use `gsdHome` for consistency.

**C. Missing-section warnings (QOBS-03) ã in core.cjs `loadConfig`:**

After successfully reading and parsing config.json, add validation:

1. Define required sections: `const requiredSections = ['quality'];`
2. For each section, check if it exists as a key in parsed config.
3. If missing, write to stderr: `process.stderr.write('Warning: config.json missing required section "quality" ã run config-ensure-section to fix\n');`
4. This is a WARNING only ã `loadConfig` continues normally and returns defaults for the missing section. Never crash, never silently ignore.

**D. Test helper extension ã in tests/helpers.cjs:**

Add a `runGsdToolsFull` function that always returns stdout and stderr:

```javascript
function runGsdToolsFull(args, cwd = process.cwd(), envOverrides = {}) {
  try {
    const result = execSync(`node "${TOOLS_PATH}" ${args}`, {
      cwd,
      encoding: 'utf-8',
      stdio: ['pipe', 'pipe', 'pipe'],
      env: { ...process.env, ...envOverrides },
    });
    return { success: true, output: result.trim(), stderr: '' };
  } catch (err) {
    return {
      success: false,
      output: err.stdout?.toString().trim() || '',
      stderr: err.stderr?.toString().trim() || '',
      error: err.stderr?.toString().trim() || err.message,
    };
  }
}
```

Export it alongside existing helpers.
