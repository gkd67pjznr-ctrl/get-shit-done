<feature>
  <name>debt.cjs — Tech Debt Tracking Module</name>
  <files>get-shit-done/bin/lib/debt.cjs, tests/debt.test.cjs, get-shit-done/bin/gsd-tools.cjs</files>
  <behavior>
    cmdDebtLog(cwd, opts, raw):
    - opts: { type, severity, component, description, logged_by, source_phase, source_plan }
    - Creates DEBT.md with header if file does not exist
    - Reads existing content to compute next TD-NNN ID via getNextDebtId()
    - Sanitizes pipe chars in ALL free-text fields: (v || '').replace(/\|/g, '/')
    - Appends one table row via fs.appendFileSync (O_APPEND atomic)
    - output({ id, logged: true }, raw, id)

    cmdDebtList(cwd, opts, raw):
    - opts: { status, severity, type } — all optional
    - DEBT.md missing → output({ entries: [], total: 0 }, raw, '[]') — never error
    - Parses all table rows via parseDebtRows(content)
    - Filters: if (opts.status) entries = entries.filter(e => e.status === opts.status) — same for severity/type
    - output({ entries, total: entries.length }, raw, JSON.stringify(entries))

    cmdDebtResolve(cwd, opts, raw):
    - opts: { id, status } — both required
    - !opts.id → error('--id required for debt resolve')
    - !opts.status → error('--status required for debt resolve')
    - Invalid status → error('Invalid status "...". Must be: open, in-progress, resolved, deferred')
    - DEBT.md missing → error('DEBT.md not found — run debt log first')
    - Finds row by escapeRegex(opts.id); uses [^|]* for each cell in regex (NOT .*)
    - Row not found → output({ updated: false, reason: `${opts.id} not found in DEBT.md` }, raw, 'false')
    - Replaces status cell and writes back via fs.writeFileSync
    - output({ id: opts.id, status: opts.status, updated: true }, raw, 'true')

    Test cases: input → expected output
    - debt log all fields → { id: 'TD-001', logged: true }
    - debt log twice → second entry is { id: 'TD-002', logged: true }
    - debt list with no DEBT.md → { entries: [], total: 0 }
    - debt list after 2 logs → { entries: [TD-001, TD-002], total: 2 }
    - debt list --status open → filters to only open entries
    - debt list --severity high → filters to only high severity entries
    - debt list --type code → filters to only code type entries
    - debt resolve --id TD-001 --status resolved → { id: 'TD-001', status: 'resolved', updated: true }
    - debt resolve --id TD-999 --status resolved → { updated: false, reason: '...' }
    - debt resolve --id TD-001 --status invalid → exits 1 with stderr error
    - debt resolve missing --id → exits 1 with error
    - debt resolve missing --status → exits 1 with error
    - description with pipe char → sanitized to slash in DEBT.md row
    - getNextDebtId on empty content → 'TD-001'
    - getNextDebtId with separator row (|---|---) → not matched (TD-\d+ pattern self-filters dashes)
  </behavior>
