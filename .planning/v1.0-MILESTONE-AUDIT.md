### Partial Requirements Detail

**EXEC-01** (partial): The executor performs targeted codebase scan before each task — this is correct. However, it scans using only the task's `<name>` and `<files>` fields, ignoring the planner's pre-populated `<code_to_reuse>` grep patterns from `<quality_scan>`. The requirement text says "targeted codebase scan" which the executor does, but the cross-phase integration intent (planner pre-loads executor) is broken.

**PLAN-01** (partial): The planner correctly generates `<quality_scan>` subsections with `code_to_reuse`, `docs_to_consult`, and `tests_to_write`. Plan-checker Dimension 9 validates these exist. However, the executor has zero references to these directives. The production side is complete; the consumption side is missing.

None. All 23 v1 requirement IDs from the traceability table are present in at least one phase VERIFICATION.md.
